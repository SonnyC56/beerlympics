# 🚀 Deploying Beerlympics (Convex + Vercel)

Two services, two homes for secrets:

- **Vercel** hosts the Next.js frontend. It needs almost nothing — just a key to reach Convex.
- **Convex** is the backend + database + file storage. **All real secrets live here** (Resend, Google/Apple OAuth, auth signing keys).
- Glue: Vercel's build command runs `npx convex deploy` first, which pushes the backend to Convex **and auto-injects `NEXT_PUBLIC_CONVEX_URL`** into the build — so you never set that by hand.

---

## 🔑 Keys to get

| Key | Where to get it | Lives on | Cost |
| --- | --- | --- | --- |
| **Convex production deploy key** | Convex dashboard → project → **Settings → Deploy Keys → Generate Production Key** | **Vercel** `CONVEX_DEPLOY_KEY` | Free |
| **Resend API key** | [resend.com](https://resend.com) → **API Keys** (verify a sending domain, or use `onboarding@resend.dev` for tests) | **Convex prod** `RESEND_API_KEY` | Free tier |
| **Google OAuth client** | [Google Cloud Console](https://console.cloud.google.com) → APIs & Services → **Credentials → Create OAuth client ID → Web application** | **Convex prod** `AUTH_GOOGLE_ID` / `AUTH_GOOGLE_SECRET` | Free |
| **Apple Sign-In** *(optional, later)* | [Apple Developer](https://developer.apple.com) → Services ID + Sign-in-with-Apple key (generate a client-secret JWT) | **Convex prod** `AUTH_APPLE_ID` / `AUTH_APPLE_SECRET` | **$99/yr** |
| Auth signing keys | Generated by `npx @convex-dev/auth --prod` (not fetched) | **Convex prod** `JWT_PRIVATE_KEY` / `JWKS` | Free |

> **Do Google first** — Apple needs the paid dev account + a fiddly secret-JWT step. Google gets you working sign-in in ~10 minutes.

## Step-by-step

### 1. Vercel
1. [vercel.com](https://vercel.com) → **Add New → Project** → import `SonnyC56/beerlympics`.
2. **Build Command** (override the default):
   ```
   npx convex deploy --cmd 'npm run build'
   ```
3. **Environment Variables** — only two:
   - `CONVEX_DEPLOY_KEY` = the Convex **production** deploy key
   - `NEXT_PUBLIC_APP_URL` = your prod URL (e.g. `https://beerlympics.vercel.app`; update once you know the real domain)
4. **Deploy.** This pushes the Convex backend to production, wires `NEXT_PUBLIC_CONVEX_URL`, and builds Next. You'll get a live URL.

> Do **not** set `NEXT_PUBLIC_CONVEX_URL` manually on Vercel — `convex deploy` injects it.

### 2. Set the Convex production secrets
Run locally (these target the project's **prod** deployment) — or set them in the Convex dashboard → **Settings → Environment Variables → Production**. `SITE_URL` is your Vercel URL.

```bash
# Auth signing keys + SITE_URL (the CLI generates the keys):
npx @convex-dev/auth --prod
#   → when prompted for the web server URL, give your Vercel URL

# Resend (email):
npx convex env set --prod RESEND_API_KEY re_xxxxxxxx
npx convex env set --prod RESEND_FROM "Beerlympics <invites@yourdomain.com>"
# optional:
npx convex env set --prod RESEND_REPLY_TO you@yourdomain.com

# Google OAuth:
npx convex env set --prod AUTH_GOOGLE_ID <client-id>
npx convex env set --prod AUTH_GOOGLE_SECRET <client-secret>
```

### 3. OAuth redirect URI (the step people miss)
In your Google OAuth client, add this **Authorized redirect URI**, using your **prod Convex** domain (in the dashboard; ends in `.convex.site`):
```
https://<your-prod-deployment>.convex.site/api/auth/callback/google
```
Apple, when you add it, uses `…/api/auth/callback/apple`.

### 4. Seed production
Prod starts empty. Load Beerlympics IV + all games:
```bash
npx convex run --prod seed:run
```
Open the site → sign in with Google → tap your avatar → **"I'm the host"** → enter the host code `HOST` (change it in **Host → Setup**).

## Env var reference

**Vercel:**
- `CONVEX_DEPLOY_KEY` — Convex production deploy key
- `NEXT_PUBLIC_APP_URL` — public site URL (for building invite links)

**Convex (production deployment):**
- `JWT_PRIVATE_KEY`, `JWKS` — auth token signing (set by `npx @convex-dev/auth --prod`)
- `SITE_URL` — your Vercel URL (OAuth redirects back here)
- `RESEND_API_KEY`, `RESEND_FROM`, `RESEND_REPLY_TO?`
- `AUTH_GOOGLE_ID`, `AUTH_GOOGLE_SECRET`
- `AUTH_APPLE_ID?`, `AUTH_APPLE_SECRET?`

---

## Gotchas
- **`SITE_URL` (Convex) and `NEXT_PUBLIC_APP_URL` (Vercel) must both equal your real prod URL** — they drive OAuth redirects and invite links. Update them if you add a custom domain.
- Real invite emails need a **verified domain in Resend**; until then use `onboarding@resend.dev` as `RESEND_FROM`.
- Dev and prod are **separate Convex deployments** with **separate env vars** — keys set on dev don't carry to prod.
- The dev deployment already has its auth keys; production needs its own (`npx @convex-dev/auth --prod`).
